# W7.2 — open-source on-chain market-maker agent

Target: `agent/` (the `calibre_agent` Python package from the W0 scaffold).
Issue: HANSEL-LI/Calibre#432. Seam 4 of `ethglobal-nyc-2026-interface.md`.

## Goal

A **standalone** market-maker agent that runs unattended from its own Dynamic
**server** wallet, reads calibre's public price (W7.1) as its prior, and acts on
the Arc `CalibreMarket` contract — with fixed size, fixed inventory cap, and a
kill-switch. Intentionally simple (not a port of the private fleet); its only
calibre input is the public price endpoint.

## On-chain venue mechanics — decided against the **real** W1.x interface

#432 says: under A-lite the trade surface is voucher-gated, so decide the exact
venue in /plan against the **real** W1.x interface. The merged contract is **W1.1
only** (`contracts/src/CalibreMarket.sol`); the EIP-712 voucher path is **W1.2
(#422), NOT yet merged**. So the primitives the contract actually exposes to a
non-resolver caller today are:

- `mint(chainMarketId, sets)` — pull `sets * usdcUnit` USDC, credit `sets` YES +
  `sets` NO (a complete set; solvent-by-construction).
- `transferShares(chainMarketId, isYes, to, amount)` — move one side's shares.
- `redeem(chainMarketId)` — after resolve, redeem the winning side 1:1 for USDC.
- views: `markets(id)` (exists, outcome), `yesBalance`/`noBalance`.

**Decision:** the agent's on-chain leg is **mint / hold / redeem** against W1.1,
NOT voucher-buys (W1.2 is unmerged — building against an interface that does not
exist would be a guess the issue forbids). This is a faithful demo maker:

- The agent reads the public YES price `p` (micro-cents, `[1,9999]`,
  `10000 = prob 1.0`).
- The fixed-spread "quote" is `[p - spread, p + spread]` micro-cents. Because
  W1.1 has no order book, the maker expresses its quote as an **inventory
  decision**: when its on-chain net exposure is under the cap it **mints** a
  fixed number of complete sets (provisioning two-sided inventory at `p`, exactly
  what an LMSR maker holds), capping notional. It does not chase fills it cannot
  place — the spread is recorded/logged as the price band it is willing to make,
  and minting is gated on `p` being inside a sane band (not near 0/1).
- After a market resolves on-chain (`markets(id).outcome != UNRESOLVED`) the
  agent **redeems** its winning shares — closing the position and recycling USDC.

When W1.2 lands, the same loop swaps `mint` for a voucher-buy with no structural
change (documented in the README as the upgrade path). This keeps the agent
runnable for the demo window **today** rather than blocked on an unmerged seam.

## Dynamic server wallet

Per Seam 4 + Dynamic's agentic-bounty docs, the agent signs from a **server
wallet** (not an embedded/user MPC wallet). Dynamic server wallets are driven
via Dynamic's wallet API; the agent needs `DYNAMIC_ENVIRONMENT_ID` +
`DYNAMIC_API_KEY` + a server-wallet id. Since a real Dynamic environment +
funded Arc testnet server wallet are **owner/booth-gated** (W8 spike: creating
the Dynamic environment and funding an Arc wallet are owner-account actions), the
signing layer is built behind a small `Signer` protocol with **two**
implementations:

1. `DynamicServerWallet` — the bounty-carrying path: provisions/loads a Dynamic
   server wallet and signs raw transactions through Dynamic's wallet API. Active
   when `DYNAMIC_API_KEY` + `DYNAMIC_ENVIRONMENT_ID` (+ optional wallet id) are
   set. The exact request shape is documented from Dynamic's server-wallet API;
   if a field is booth-confirmed-only it is isolated to this class.
2. `LocalKeySigner` — a `web3`/`eth-account` local-key signer (the same
   primitive the merged SDK uses) for running the agent against Arc testnet
   without a Dynamic account. Active when only `AGENT_PRIVATE_KEY` is set.

This satisfies the bounty (server-wallet path is the default when credentials
exist) **and** keeps the artifact runnable + testable offline. The choice is
config-driven and logged at boot.

## Files to touch (all under `agent/`)

- `agent/src/calibre_agent/__init__.py` — drop the W0 SCAFFOLD note; export the
  public surface (`AgentConfig`, `run`, `MarketMaker`, signer classes).
- `agent/src/calibre_agent/config.py` — `AgentConfig` dataclass + `from_env()`:
  Arc RPC/chain/contract, public-API base, market id, fixed size, spread band,
  inventory cap, poll interval, dry-run, kill-switch file, signer credentials.
- `agent/src/calibre_agent/price.py` — `PriceFeed`: GET
  `{base}/markets/public/{id}/price` → `(price_yes, as_of)`; httpx; raises on
  non-200 (404 = market not open) so the loop can back off. No authed calls.
- `agent/src/calibre_agent/contract.py` — minimal hand-written ABI for
  `mint`/`transferShares`/`redeem`/`markets`/`yesBalance`/`noBalance` matching
  `CalibreMarket.sol` (the SDK only encodes create/resolve), plus a `usdc`
  ERC-20 `approve`/`allowance`/`balanceOf` slice. Build/sign/send via the Signer.
- `agent/src/calibre_agent/signer.py` — `Signer` protocol +
  `LocalKeySigner` + `DynamicServerWallet`.
- `agent/src/calibre_agent/maker.py` — `MarketMaker`: one decide-and-act step
  (read prior → band check → inventory check → mint or redeem), with the risk
  bounds. Pure-ish: takes feed + contract + config, returns an action record.
- `agent/src/calibre_agent/loop.py` — `run()`: the unattended loop (poll →
  step → sleep), kill-switch check each tick, structured logging, bounded by the
  inventory cap so a bug is bounded (success-criterion: cap funding).
- `agent/src/calibre_agent/__main__.py` — `python -m calibre_agent` entrypoint
  reading `AgentConfig.from_env()`.
- `agent/pyproject.toml` — add deps (`httpx`, `web3`, `eth-account`) + a
  `calibre-agent` console script + a `[dependency-groups]`/`test` extra
  (`pytest`); fill version.
- `agent/README.md` — usage, the "prior is a public read / strategy is
  intentionally simple" statement, the risk bounds + kill-switch, the
  server-wallet vs local-key choice, and the W1.2 voucher upgrade path.
- `agent/tests/` — unit tests with a fake feed + fake contract/signer (no
  network, no chain): band gate, inventory cap, kill-switch, redeem-after-resolve,
  dry-run, and config-from-env. Plus the env placeholders in root `.env.example`
  already cover `DYNAMIC_*`; add `AGENT_*` keys there.

## Risk bounds (success criterion: a bug is bounded)

- `size` — fixed sets per mint (default small).
- `inventory_cap_sets` — never mint beyond this many net sets on the market;
  the loop refuses to mint past it (caps total USDC at risk).
- `band` — only mint when `band_lo <= price_yes <= band_hi` (skip degenerate
  near-0/near-1 prices).
- `kill_switch_file` — if the file exists, the loop halts new actions (operator
  stop without killing the process); also a max-iterations guard for demos.
- `dry_run` — log intended actions without sending tx (default ON until funded).

## Risks / notes

- **Voucher path unmerged** — building against W1.2's voucher `buy` would be a
  guess; we use the real W1.1 mint/redeem surface and document the swap.
- **Dynamic server-wallet credentials** are owner/booth-gated — the agent ships
  with `.env.example` placeholders and a working local-key fallback so the
  artifact runs offline and on Arc testnet without a Dynamic account.
- No private imports, no authed calls — checkable from the public repo (the only
  calibre input is the W7.1 public price endpoint).
