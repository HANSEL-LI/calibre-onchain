# W7.2 voucher swap — switch the agent's mint leg to W1.2 voucher-buys (#444)

Target: `agent/` (the `calibre_agent` package). Issue: HANSEL-LI/Calibre#444 (p3).
Source contract: `contracts/src/CalibreMarket.sol` (W1.1 + the now-merged W1.2
voucher extension, #422). Plan W1.2: `docs/plans/W1.2-voucher.md`.

## Goal

Swap the agent's on-chain **MINT** leg for a **W1.2 EIP-712 voucher-buy** against
calibre's standing-counterparty contract, with **no change to the strategy
(`maker.py`) and the loop control flow (`loop.py` run()/guards)**. Only
`MarketClient` gains the voucher call + the one MINT-branch wiring line in
`loop.step` flips from `client.mint(...)` to `client.buy(...)`.

## The real merged contract surface (verified against `CalibreMarket.sol`)

- `buy(Quote q, uint256 cost, bytes sig)` — **3 args** (not the issue skeleton's
  `buy(quote, sig)`). `cost` is the actually-charged USDC (6-dec), `cost <= q.maxCost`.
- `struct Quote { uint256 marketId; address buyer; uint8 side; uint256 size;
  uint256 maxCost; uint256 nonce; uint256 expiry; }` — buyer == msg.sender;
  side 0=NO, 1=YES.
- `_QUOTE_TYPEHASH = keccak256("Quote(uint256 marketId,address buyer,uint8 side,uint256 size,uint256 maxCost,uint256 nonce,uint256 expiry)")`.
- Domain: `name="CalibreMarket"`, `version="1"`, `chainId=block.chainid`,
  `verifyingContract=address(this)`. `hashQuote` = `keccak(0x1901 ‖ domainSep ‖ structHash)`.
- `nonces(buyer)` view (per-buyer monotonic), used to fill `q.nonce`.
- 65-byte (r,s,v) sig; EIP-2 high-s rejected (so `encode_typed_data` + `Account.sign`
  canonical-low-s output is fine).

**Who signs:** the agent is the **buyer** (`msg.sender == q.buyer`). The voucher is
signed by **calibre's backend signer (W3.1)** — `voucherSigner` is a calibre key,
not the agent's. So the agent must *obtain* a signed voucher, then submit it under
its own signing identity (Dynamic server wallet / local key) as the transaction
sender. Two roles, two keys: tx-sender (agent) vs voucher-signer (calibre).

## Design — a `VoucherSource` behind one protocol (mirrors the `Signer` pattern)

New `agent/src/calibre_agent/voucher.py`:

- `VoucherRequest` / `SignedVoucher` dataclasses (quote dict + cost + 65-byte sig).
- `VoucherSource` Protocol: `fetch(market_id, side, size, buyer) -> SignedVoucher`.
- `CalibreVoucherClient` — the **production path**: POSTs the buy intent to a
  calibre quote endpoint (W3.1, in the private app) and returns the LMSR-priced,
  backend-signed voucher. The exact endpoint/field names are isolated to this
  class (booth/owner-gated, documented from the W3.1 interface), exactly like
  `DynamicServerWallet`. Selected when `CALIBRE_VOUCHER_API_BASE` is set.
- `LocalVoucherSigner` — the **offline/testnet fallback**: signs the quote locally
  with `AGENT_VOUCHER_SIGNER_KEY` via `eth_account.encode_typed_data` against the
  frozen domain/struct, so the artifact runs + tests offline (mirrors
  `LocalKeySigner`). Computes `maxCost`/`cost` from the public prior × size × 1.01.
  Selected when only `AGENT_VOUCHER_SIGNER_KEY` is set.
- `build_voucher_source(config, *, default_signer_address)` — picks the source;
  raises if neither configured (parallels `build_signer`).

`contract.py`:
- Add the `buy` + `nonces` ABI fragments (byte-matched to the merged contract).
- `MarketClient` gains `voucher_source` (optional) + `chain_id` is already held;
  add `def buy(self, market_id, side, size) -> str`: read `nonces(buyer)`, ask the
  voucher source for a `SignedVoucher` (passing the agent address as `buyer` + the
  on-chain nonce), assemble the `buy(q, cost, sig)` tx, send via the Signer. Keeps
  `mint`/`redeem`/`ensure_allowance` for the redeem leg + USDC approve (the buyer
  still approves USDC so the contract can pull `cost`).

`maker.py`: **unchanged** — still emits `ActionKind.MINT` with `sets`/band. The
"MINT" action name is the maker's intent ("provision a position at the prior"); the
venue it maps to is the loop's concern. (Renaming the enum would ripple through the
pristine `test_maker.py` for zero functional gain — parsimony.)

`loop.py`: the MINT branch swaps `client.mint(config.market_id, action.sets)` →
`client.buy(config.market_id, side=OUTCOME_YES, size=action.sets)` and keeps the
USDC `ensure_allowance` (the buyer still needs allowance for the `cost` pull). The
run()/kill-switch/dry-run/max-iterations/inventory-cap control flow is **untouched**.
Side choice for the demo maker: buy YES `size` (the maker takes the YES side at the
prior; the band/cap rails are unchanged). Documented in Decisions.

## EIP-712 digest parity (the #443-flagged on-chain-break risk)

Verified offline that `eth_account.encode_typed_data(domain, {Quote: ...}, message)`
produces a digest **byte-identical** to a Solidity-faithful manual
`keccak(0x1901 ‖ keccak(abi.encode(DOMAIN_TYPEHASH, …)) ‖ keccak(abi.encode(QUOTE_TYPEHASH, marketId, buyer, side, size, maxCost, nonce, expiry)))`
— same typehash strings, same field order, same framing as `CalibreMarket.hashQuote`.
This equality is frozen into a regression test (`test_voucher.py::test_digest_matches_contract_hashquote`)
so any future drift in field order / domain fields fails loudly offline. (forge is
not installed in this env, so the manual-abi reference — line-for-line the contract's
`hashQuote` body — is the parity oracle; noted under Decisions/BLOCKED.)

## Decisions (rationale)

- **`buy(q, cost, sig)` 3-arg, not the issue's `buy(quote, sig)`** — the merged
  contract takes `cost` as a separate arg (the ≤ maxCost charge). Built against the
  real ABI, not the skeleton.
- **Agent obtains a voucher; calibre signs it** — `voucherSigner` is a calibre key,
  not the agent's. Production = `CalibreVoucherClient` HTTP; offline/testnet =
  `LocalVoucherSigner` with a configured key (mirrors `LocalKeySigner`).
- **`maker.py` MINT enum kept** — it is the maker's *intent*; the venue mapping is
  the loop's job. Renaming ripples pristine tests for no functional gain.
- **Buy the YES side at the prior** for the demo maker — keeps a single, legible
  inventory leg under the existing band/cap rails (NO-side / two-sided making is out
  of scope for this p3 swap).
- **Manual-abi digest oracle** (forge absent) — the parity test reproduces the
  contract's `hashQuote` body exactly in `eth_abi.encode`; equality is frozen.

## Safety rails preserved (success criteria)

- `dry_run` default ON — the loop still logs intent without sending in dry-run.
- inventory cap — the maker still gates MINT on `net_sets + size <= cap`; the loop
  refuses past-cap. (Voucher buys credit only the bought side; `net_sets` stays a
  conservative `min(yes,no)` bound.)
- kill-switch + max-iterations — `loop.run` control flow untouched.

## Risks / BLOCKED

- **Live voucher endpoint (W3.1) shape** — the calibre quote/sign endpoint lives in
  the private app and is owner/booth-gated. `CalibreVoucherClient`'s request/response
  shape is documented from the W1.2 interface; real wiring is BLOCKED on the private
  endpoint. The offline `LocalVoucherSigner` keeps the artifact runnable + tested.
- **forge not installed** — on-chain `hashQuote` call-parity can't run here; the
  manual-abi oracle stands in (see Decisions). Re-verify against a deployed contract
  at W1.3.
