# W1.3 — scripted end-to-end voucher proof (Saturday-noon custody checkpoint, #423)

> Target repo: **calibre-onchain**. This is the umbrella's custody gate
> (briefing §3): a green run keeps custody at **Model A-lite**; a red run
> degrades W2.3 (#427) and W3.1 (#424) to **Model B**. The proof must exercise
> the *whole* merged contract surface end-to-end as a single scripted runner,
> not a unit test.

## What we are proving

`deploy(usdc, resolver) → seedInventory/mint → voucher-signed buy(quote, sig)
with msg.sender == buyer → resolve(marketId, outcome) by the resolver →
redeem winners 1:1 → assert final balances + solvency (contract backing ==
outstanding redeemable, drains to exactly 0)`.

The same script structure must later run **unchanged** against live Arc testnet
(chainId 5042002, RPC `https://rpc.testnet.arc.network`, 6-decimal ERC-20 USDC)
by swapping the `--rpc-url` + a funded `--private-key`. That live run is
owner/booth-gated and is **not** attempted here.

## Canonical run = local anvil (deterministic, no funding)

Per the prompt, the canonical scripted proof runs against a **local
Foundry/anvil chain**: deterministic, no testnet faucet, no funded key needed.
A `forge script` broadcasts real transactions to anvil so every step is a true
on-chain call (deploy, seed, buy, resolve, redeem) rather than an in-memory
`Test` assertion. This is the distinction from the existing
`CalibreVoucher.t.sol`: the e2e proof is a *scripted, broadcastable round-trip*,
the success criterion #423 names.

### Local vs live in one script

- **USDC:** local runs deploy the existing `test/mocks/MockUSDC.sol` (6-dec,
  matches the Arc ERC-20 interface) and mint to the actors — no faucet. Live
  runs read `USDC_ADDRESS` from env (the real Arc USDC) and skip the deploy +
  mint, expecting the funded key to already hold USDC. Selected by whether
  `USDC_ADDRESS` is set (mirrors `Deploy.s.sol`'s `vm.envAddress`).
- **Keys:** local uses three deterministic anvil keys (resolver/counterparty/
  buyer) + the voucher signer as a known test key, so `vm.sign` can produce a
  real EIP-712 signature the on-chain `buy` verifies. Live swaps in the funded
  broadcaster.
- **Assertions:** `require`-based, so a failed invariant aborts the broadcast
  with a non-zero exit — the script's pass/fail *is* the checkpoint verdict.

## Test / run command (canonical, local anvil)

```bash
cd contracts
# terminal A — local chain
anvil
# terminal B — broadcast the full round-trip against it
forge script script/EndToEnd.s.sol:EndToEnd \
  --rpc-url http://127.0.0.1:8545 --broadcast -vvv
```

A self-contained one-shot used for this proof (anvil in background, run, assert

## Decisions

- **Local anvil is the canonical proof; live Arc is the swap.** The prompt is
  explicit: deterministic local run as the scripted proof, structured to swap
  RPC+key for live. Avoids faucet dependence and is reproducible in CI/booth.
- **Reuse `MockUSDC` for local, real USDC via env for live.** Don't add a second
  mock; the deployed-token path is selected by `USDC_ADDRESS` presence, exactly
  mirroring `Deploy.s.sol`.
- **`require` assertions inside the script** (not a `Test` harness) so the
  broadcast itself is the pass/fail signal — a red run reverts and the orchestrator
  sees a non-zero exit. This is what makes it a *checkpoint*, not just coverage.
- **Exercise both YES and NO buys + counterparty residual redeem** so the
  drain-to-zero solvency assertion covers the full set lifecycle, not just one
  side — strictly stronger than a single-side happy path.

## Risks

- **Foundry not on PATH** in this environment — resolved by prepending
  `~/.foundry/bin`; CI uses `foundry-toolchain@v1` (unchanged).
- **Local EVM ≠ Arc** (W8 §3: blocklist/precompile, native-vs-ERC20 decimals).
  The script proves *logic*; the README flags that only Arc's RPC reproduces
  Arc-specific transfer reverts, so the live run remains the final confirmation.
  Checkpoint scope is the logical round-trip, which local anvil proves faithfully.
